# API

> **Refs**: [ARCHITECTURE](./ARCHITECTURE.md) · [SECURITY](./SECURITY.md) · [SEQUENCE_DIAGRAM](./SEQUENCE_DIAGRAM.md)

route handler 엔드포인트, 권한, 상태 코드를 정의한다.

## 공통 규칙
- 모든 API 로직은 `app/api/` route handler에서만 처리한다.
- 응답은 `{ data }` 또는 `{ error }` 형태로 통일한다.
- 인증/인가 정책은 [SECURITY](./SECURITY.md) 참조.

## 엔드포인트

### `{METHOD} /api/{경로}`
| 항목 | 내용 |
|---|---|
| 설명 | {무엇을 하는가} |
| 권한 | {public / 로그인 / 소유자 / admin} |
| 요청 | {body/query 스키마} |
| 응답 | {성공 시 반환 형태} |
| 상태 코드 | 200 / 400 {검증 실패} / 401 {미인증} / 403 {권한 없음} / 404 / 500 |

### `{METHOD} /api/{다른 경로}`
| 항목 | 내용 |
|---|---|
| 설명 | {무엇을 하는가} |
| 권한 | {정책} |
| 요청 | {스키마} |
| 응답 | {형태} |
| 상태 코드 | {목록} |

## 에러 코드 규약
| 코드 | 의미 | 사용 시점 |
|---|---|---|
| 400 | 입력 검증 실패 | {} |
| 401 | 미인증 | {} |
| 403 | 권한 없음 | {} |
| 404 | 리소스 없음 | {} |
