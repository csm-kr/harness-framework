# 상태 모델

> **Refs**: [DB](./DB.md) · [ARCHITECTURE](./ARCHITECTURE.md)

앱 레벨(휘발성) 상태와 서버 영속 상태를 구분한다. 저장 스키마는 [DB](./DB.md).

## 앱 레벨 상태 (클라이언트, 휘발성)
| 상태 | 위치 | 수명 | 비고 |
|---|---|---|---|
| {예: 모달 열림 여부} | {useState} | {컴포넌트} | |
| {예: 폼 입력값} | {useState/form} | {제출까지} | |
| {예: 테마} | {localStorage} | {브라우저} | |

## 서버 영속 상태 (DB)
| 상태 | 모델 | 갱신 시점 |
|---|---|---|
| {예: 사용자 프로필} | {User} | {수정 시} |
| {예: 작업 진행도} | {Phase/Step} | {step 완료 시} |

## 파생/캐시 상태
| 상태 | 도출 방식 | 무효화 시점 |
|---|---|---|
| {예: 진행률} | {완료/전체} | {step 상태 변경} |

## 동기화 규칙
- {예: 서버가 단일 진실 공급원(SSOT). 클라이언트는 낙관적 업데이트 후 재검증}
