# 이슈 · 반복 문제

> **Refs**: [RULES](./RULES.md) · [ADR](./ADR.md)

**개발/실행 중에 발생한 이슈**(에러, 막힌 지점, 버그, 반복되는 실수)를 기록한다. step 실행 중 `error`/`blocked`가 나거나 같은 실수를 반복하면 여기에 남긴다. (step의 `error_message`·`blocked_reason`이 1차 소스다.)

> **승격 규칙**: 같은 문제·실수가 **3회 이상 반복되면** 일회성 이슈가 아니라 구조적 문제다. 아래 "반복 이슈 추적"에서 발생 횟수를 세고, 3회에 도달하면 [RULES](./RULES.md)의 불변 규칙으로 **승격**한 뒤 이 표에서는 닫는다.
